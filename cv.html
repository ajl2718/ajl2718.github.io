CV

My background

I studied maths and physics, completing my PhD in mathematics in 2015. Since then I have been working in various data science and science roles. I have led over 15 data science projects in business, government and academia, with a recent focus on clinical data science.

My current projects involve application of machine learning and statistical techniques to linked primary care data for cancer research. This is a complex area in terms of the data and domain knowledge required, and also involves a fair amount of bureaucratic wrangling (definitely the less fun part of the job).

My technical expertise ranges from data cleaning and wrangling (Pandas, Polars, DuckDB, Arrow), to machine learning model development (Pytorch, Keras), data visualisation (seaborn, ggplot), deployment (Azure) and, recently, fine-tuning LLMs for faciliating data science workflows.

A large part of my current role and previous role involves communicating and translating between domains, for example working with clinicians to refine ideas so that they can be translated into data science problems. This is challenging but also rewarding and a good source of ideas for data science applications.

Open source projects

whereabouts is an open-source Python package for geocoding at scale (1000s of addresses per second). The geocoding algorithm (based on this paper) is implemented in SQL using DuckDB. This means you no need to setup a database server and you can install everything using pip.
